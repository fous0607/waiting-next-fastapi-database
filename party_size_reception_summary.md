# 다이닝 모드 및 인원수 설정 구현 완료 보고서

## 변경 사항 요약

### 1. 다이닝 모드 (선불/후불) 구분
- 매장 설정에 **"세부 운영 방식 (Service Type)"** 추가 (일반 식당 vs 카페/픽업).
- 방식에 따라 대기자 목록 버튼이 "호출/입장" 또는 "준비 완료/수령 완료"로 자동 변경됩니다.

### 2. 인원수 설정 개선 (Party Size Configuration)
기존의 모호했던 "분류(필수/선택)" 설정을 명확한 **"유형(Type)"** 설정으로 변경하였습니다.
- **인원수 포함 (Standard)**: 실제 대기 인원수에 포함됩니다. (예: 성인, 어린이)
- **옵션/비포함 (Option)**: 대기 인원수 합계에서는 제외되고 상세 정보에만 표시됩니다. (예: 아기의자, 유모차)

### 3. 대기접수 데스크 (Reception Desk) 연동
공용 대기접수 페이지(`/entry/[store_code]`)가 매장 설정을 올바르게 반영하도록 수정하였습니다.
- 매장의 **"인원수 입력 사용"** 설정이 켜져 있으면, 접수 화면에 인원수 입력 칸이 자동으로 나타납니다.
- 설정한 카테고리(성인, 어린이 등)별로 인원수를 입력받습니다.
- **총 대기 인원**은 "인원수 포함(Standard)" 유형의 합계로 자동 계산되어 저장됩니다.

## 검증 결과

### 빌드 상태
- `npm run build`: **성공 (Success)**

### 테스트 방법 (인원수 설정)
1. **설정 > 일반** > **외식 모드 특화 기능**으로 이동합니다.
2. "인원수 입력 사용"을 체크합니다.
3. "인원수 카테고리 구성"에서 '성인', '아동', '유아의자' 등을 추가합니다.
4. '성인', '아동'은 **"인원수 포함"**으로, '유아의자'는 **"옵션/비포함"**으로 설정하고 저장합니다.
5. **대기접수 데스크** (QR코드 또는 링크)를 엽니다.
6. 인원수 입력란이 정상적으로 표시되는지, 총 인원수가 '성인+아동'으로만 계산되는지 확인합니다.
